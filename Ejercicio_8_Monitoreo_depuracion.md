Ejercicio 8 – Monitoreo y Depuración

En este ejercicio se introdujo un error intencional dentro de un workflow
de GitHub Actions utilizando un comando inexistente.

Se habilitó el modo de depuración mediante la variable ACTIONS_STEP_DEBUG,
permitiendo visualizar mensajes de debug, warnings y errores.

También se utilizaron comandos para inspeccionar:
- Directorios y archivos
- Variables de entorno
- Procesos del sistema
- Espacio en disco

Esto permite identificar rápidamente la causa del fallo y mejorar
la capacidad de diagnóstico de workflows en CI/CD.
